## Taks 1

### Есть решения для API Gateway:
- Kong                 — производительный Gateway на базе NGINX с открытым исходным кодом;
- Express Gateway      — легковесное решение на Node.js;
- Amazon API Gateway   — полностью управляемый сервис от AWS;
- Azure API Management — аналогичное решение от Microsoft;

# Express Gateway. Возможности:
 - Логирование;
 - Маршрутизация;
 - Аутентификация;
 - Сбор данных;
 - Кэширование;

# Kong. Возможности:
- соблюдение правил безопасности (например, проверка JWT);
- ограничение запросов и ограничение количества запросов;
- выполнение плагинов и middleware;
- проксирование трафика и балансировка нагрузки;

### Возможно, подойдёт Kong:
Бесплатный. Есть возможность расширить функционал.

## Task2

### RabbitMQ
    Удобен там, где сложная маршртизация и уравление очередями, но при этом небольшой объём данных .Гарантирует высокую надёжность доставки сообщений.

Преимущества:
- Удаление сообщений из очереди — после получения потребителями сообщение удаляется из очереди, что позволяет одно и то же сообщение обработать только одним потребителем;
- Гибкая маршрутизация;
- Брокер сам активно отправляет сообщения пользоветелям, подписанным на очереди;

### Apache Kafka
    Работает с большими данными, как рыба в воде. Раскидывает сообщения по папкам и раздаёт, если нужно.

Преимущества:
- отказоустойчивость;
- масштабируемость;
- возможность хранения данных;

### Redis
    Высокая скорость доступа к данным, так как данные лежат в ОЗУ. Но из-за способа хранение данных, низкая надёжность, так как данные могут быть утеряны при сбоях.

Преимущества:
- Сверхнизкая задержка — задержки измеряются микросекундами.
- Широкие возможности кеширования — быстрый доступ к часто запрашиваемым данным, хранение сессий, счётчиков, рейтингов.
